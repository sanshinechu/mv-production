# 工作日誌 2026-06-17

## ✅ 今天完成

### mv-production 線上網頁：多專案切換功能

幫 `sanshinechu.github.io/mv-production/`（純 HTML/JS + Firebase RTDB）補上「登入後可切換不同專案」的完整功能。

- **登入後側邊欄新增專案選單**：自動載入使用者存過的所有專案，預設開啟最近更新的一個
- **切換**：選單選一個專案 → 內容（歌詞/主題/曲風/情緒）載回表單
- **新增 / 刪除**：「＋ 新增專案」清空表單開新的；「刪除」移除目前專案（二次確認）
- **儲存邏輯改寫**：選定專案時是「更新」該筆，新專案才「建立」，不再每次重複新增；存檔成功顯示「✓ 已建立/已更新」
- **修掉舊 bug**：移除造成 Step_01 重複顯示的重複 HTML 區塊；補存先前漏掉的「歌詞」欄位

### 依使用者需求調整表單

- 歌詞框改為整排全寬置頂，修正移除欄位後的版面對齊

### 重大除錯：儲存無反應

使用者回報「儲存專案按了沒反應、也沒錯誤」。診斷出根因並修復：

- Realtime Database 實際在 **asia-southeast1**，但 `firebase-config.js` 缺 `databaseURL`，SDK 預設連美國端（不存在）→ 寫入靜默卡住
- 補上正確 `databaseURL` 後，**在線上用真實帳號實測整輪通過**（寫入 65ms、UI 存檔→選單顯示→刪除全正常），測試資料已清乾淨
